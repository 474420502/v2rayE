# v0.1.4 发布说明

发布日期：2026-05-10

`v0.1.4` 是一个小版本稳定性发布，重点不是扩展新页面，而是把最近终端主路径里最容易影响实际使用的问题先收紧。

## 重点变化

- 运行时切换稳定性增强
  - `native` 服务进一步约束并发启动、重启和运行时切换路径。
  - 节点切换、路由保存、配置变更时，核心重启与状态切换更接近串行执行，减少“配置已写入但运行态尚未跟上”的短暂偏差窗口。

- TUI 页面切换更稳
  - 页面切换和语言切换现在通过专门的 UI 切换锁串行化。
  - 在后台轮询和 SSE 刷新存在时，页面导航和 UI 重标记不再那么容易互相踩踏。

- 删除节点增加真正二次确认
  - 删除弹窗会先显示节点摘要。
  - 只有在对话框中输入 `DELETE` 后，删除动作才会真正执行。
  - 这次改动主要针对终端环境下误触 Enter 导致的误删风险。

- 导入重复节点收敛
  - 连续点“导入剪贴板”时，完全相同的节点不会再被重复追加到列表中。
  - 去重规则只针对“完全相同”的节点。
  - 如果只是 IP/地址不同，即使其他字段相同，也仍然视为不同节点，保留两条记录，便于节点迁移和多入口场景。

## 对用户的直接影响

- TUI 中删除节点需要多一步确认，但安全性更高。
- 连续重复导入同一个剪贴板节点时，列表不会再被同一条记录刷满。
- 使用同一套参数、但不同 IP 的节点时，不会被错误合并。
